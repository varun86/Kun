# Kun v0.2.26

这一版是 v0.2.25 之后的一次重要功能与稳定性更新。重点是推出 Kun Extension 开放平台和 PPT Master，扩展 SVG 白板、项目设计系统、文件预览、模型与 Subagent 配置能力，同时对 Kun Runtime 的安全边界、资源占用和退出清理做了一轮系统性加固。

### Kun Extension 开放平台

- 新增 `.kunx` 扩展的安装、侧载、启用、禁用和管理能力，可以从设置中的扩展管理入口统一操作。
- 扩展可以为 Kun 提供工作台页面、侧栏入口、后台 Node 服务、Agent、工具、自定义模型 Provider 和账号接入。
- Extension API v1 提供 Manifest Schema、权限确认、独立 Extension Host、受控工作台贡献点、Webview、Direct DOM 和内容脚本等能力边界。
- 新增 `npm create kun-extension` 脚手架、扩展打包与测试工具，以及多个可运行的示例项目。
- 补充完整的中英文扩展文档，覆盖快速开始、Manifest、生命周期、权限与资源、Agent 与工具、Provider 与账号、打包发布、版本迁移和故障排查。

### Write 与 PPT Master

- Write 工具栏新增“使用 PPT Master 生成 PPT”，可以从当前 Markdown 文档发起演示文稿生成流程。
- 生成前会先确认幻灯片大纲、页数、目标受众和视觉方向，确认后生成原生可编辑的 `.pptx`。
- 生成结果会保存到工作区的 `presentations/` 目录，来源 Markdown 不会被修改。
- PPT Master 使用固定版本和隔离 Python 环境，补强首次安装、任务续跑、重启恢复和异常重试流程。
- 改进 Write 文档状态、媒体与引用布局、图片粘贴、保存协调和工作区持久化，降低切换文件或退出时丢失改动的风险。

### Design、SVG 白板与设计系统

- 新增 SVG 白板产物，可在 Design 中生成、预览、继续修改和持久化 SVG 设计稿。
- 支持读取项目中的 `DESIGN.md`，把颜色、排版、组件和主题样例呈现为可浏览的设计系统画板。
- 改进流程图、圆角矩形、图片标注、画布复制粘贴和实时 shape 操作，并新增画布图片导出。
- 工作区图片和 SVG 文件现在可以直接进入预览，不再只能从外部应用打开。
- 修复历史画板加载、Design 活跃线程输入、白板重新发送路由和多工作区切换时的状态一致性问题。

### 模型、Composer 与 Agent

- 新增 ChatGPT Subscription GPT-5.6 Lite 支持，并支持 Codex 原生图片生成模型。
- 新增 Subagent 设置界面，可以集中配置子代理模型、提示词、工具、权限和运行方式。
- Composer 将模型选择和推理强度拆分为独立控制，切换模型时更清楚，也更适合窄窗口布局。
- 结构化用户输入新增多选支持，并改进紧凑 Composer 中的输入回复处理。
- 支持工作区级 MCP 配置和项目 `.kun/skills` 目录，让项目可以携带自己的工具连接与技能说明。
- 修复自动路由和 Agent 配置中的 Provider 隔离问题，避免指定模型意外落到错误的供应商。

### 文件、PDF 与工作区体验

- 扫描版 PDF 新增 OCR fallback，在常规文本提取为空时仍可识别页面内容。
- 改进文件创建、重命名、移动、预览和打开操作，减少文件树与实际工作区状态不同步的情况。
- 支持在线程之间移动项目工作区，并进一步明确移动范围和侧栏操作语义。
- 长会话消息时间线改为按需加载，降低大型线程初次渲染的开销。
- 更新安装、窗口关闭和无效工作区根目录增加更稳妥的处理与 fallback。

### Runtime 安全与稳定性

- 加强工具调用、MCP、Skills、Agent SDK、代码审查和语言服务器的路径约束、权限确认与进程隔离。
- Web Fetch 会阻止 SSRF 重定向；运行服务、附件、工作区指令、生成路径和线程存储增加更严格的边界检查。
- 对附件上传、模型流解析、SSE 回放、后台 Shell、并发回合、诊断缓存和临时输出设置资源上限，降低长时间运行后的内存与磁盘压力。
- 收紧运行时数据、附件、日志和产物目录的文件权限，并对错误与日志中的 Provider URL、凭据等敏感信息进行脱敏。
- 加固线程创建、删除、取消、压缩、恢复和历史记录提交的一致性，减少并发操作、过期历史修复和事件回放导致的状态竞争。
- 应用退出时会更完整地终止 Agent、Subagent、工作流、IM、定时任务、语音识别、LSP 和后台进程，避免残留任务继续占用资源。
- 重构 Kun Agent Loop、模型协议适配、工作流执行、持久化和 Runtime 生命周期边界，并补充回归测试与可观测性覆盖。

### 升级说明

- 从 `v0.2.25` 升级可直接通过 GUI 更新，现有工作区、Provider、MCP、Skills 和会话数据不需要手动迁移。
- PPT Master 首次使用时会按需准备隔离运行环境，需要本机已安装 Python 3.10 或更高版本。
- `.kunx` 扩展可能申请 Node、Direct DOM 等高权限能力，请只安装可信来源的扩展，并在安装时仔细检查权限提示。
- 项目级技能可以放在 `.kun/skills`；工作区 MCP 配置会与现有全局配置共同生效。

### 总结

v0.2.26 把 Kun 从一个功能完整的桌面 Agent 工作台继续推向可扩展的平台：PPT Master 让 Markdown 可以直接进入演示文稿交付，SVG 白板和 `DESIGN.md` 让设计资产更容易沉淀，Extension API 则让第三方能力能够以明确的权限和生命周期接入 Kun。与此同时，这一版也花了大量工作收紧 Runtime 的资源与安全边界，让这些新能力更适合长期、复杂的真实任务。
